# Project Scope

## 1. 代码层面的最终目标

本仓库最终要实现的不是“把文件导入数据库”，而是基于已经入库的航空人机多源时序数据，构建一条可复用的下游融合链路：

1. 从 InfluxDB 读取生理数据与飞机侧时序数据，从 MySQL 读取业务元数据与字段映射。
2. 将不同来源、不同频率、不同时间语义的数据整理为统一的研究输入对象。
3. 实现双流连续时间表示与对齐方法，用于承接“人”和“机”两个不同动力学域。
4. 实现带物理一致性约束的时间对齐机制。
5. 实现带因果方向约束的跨模态融合结构，重点是“航电/事件影响生理状态”的单向逻辑。
6. 输出标准化融合特征矩阵和可解释中间态，用于空中失能风险分析、认知负荷评估、飞行事件复盘等下游任务。
7. 支撑离线实验、对比实验、消融实验，以及后续近实时特征生成原型。

## 2. 当前范围边界

### In scope

- InfluxDB / MySQL 数据访问层
- 统一 schema 与样本组织
- 连续对齐模型原型
- 因果融合模型原型
- 标准化特征导出
- 评测、可视化与案例复盘
- 面向后续服务化的推理接口预留

### Out of scope

- 重写历史接收器或重新实现上游导入链路
- 在本仓库中保存原始实采大文件
- 一次性建立覆盖 2000+ 总线文件的全量静态模型
- 把 notebook 或临时脚本当作正式实现
