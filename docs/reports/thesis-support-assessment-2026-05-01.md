# Thesis Support Assessment

更新时间：2026-05-01

## 1. 目的

本文件不是新的实验报告，而是把当前仓库里已经完成的 `E/F/G/H/I` 与选题报告中的论文主线做一次对齐，回答三个问题：

1. 当前实验已经推进到论文中的哪一步。
2. 现有证据能否分别支撑“`双流对齐`”和“`因果融合`”两个部分。
3. 如果目标是让论文更有说服力，下一步最该补什么，而不是继续泛化地“多跑一些实验”。

## 2. 选题报告中的论文主线与当前代码映射

根据选题报告，论文的核心可以拆成三段：

### 2.1 双流对齐

对应当前仓库的：

- `阶段 E`：双流连续对齐最小原型
- `阶段 F`：物理一致性约束对齐
- `阶段 H`：对齐后标准化导出与稳定读取

通俗理解：

- `E` 解决“能不能把两条不同时间精度、不同采样机制的数据放到同一个连续时间参考轴上”
- `F` 解决“这种对齐是不是符合飞行物理与生理变化常识”
- `H` 解决“对齐后的结果是不是已经变成一个稳定、可复用、可下游消费的标准资产”

### 2.2 因果融合

对应当前仓库的：

- `阶段 G(min)`：最小非对称因果融合
- `阶段 H`：融合特征固化与导出
- `阶段 I Phase 2`：基于真实 sortie 的 case study / ablation / dual-pilot 对照

通俗理解：

- `G` 解决“生理状态能不能按选题报告里的单向物理因果逻辑去看历史航电事件，而不是双向乱注意”
- `Phase 2` 解决“这种融合是不是至少在真实 sortie 上能给出可解释的注意力和干扰敏感性”

### 2.3 下游任务验证

对应当前仓库的：

- `阶段 I Phase 1/3`：UAB / NASA 公开数据 benchmark
- `阶段 I 增强实验`：`MulT / ContiFormer` 横向对照

通俗理解：

- 这部分的作用不是重新证明你的主管线“被实现了”
- 而是回答“与常见 benchmark / 对照模型相比，你的方法路线值不值得继续保留为论文主线”

## 3. 当前实验到底做到哪一步了

当前不是“还在开发系统”，而是已经进入：

`论文第三段：系统级验证、案例分析、对比与消融`

更具体地说：

1. `双流对齐` 这部分已经不是概念设计，而是完成了真实 sortie 上的实现、对齐、物理约束、导出与读取闭环。
2. `因果融合` 这部分也已经不是空架子，而是完成了最小因果融合、真实 sortie 导出、注意力与事件贡献解释、bundle-only 消融。
3. `公开数据 benchmark` 也不是待开始，而是已经完成 classical baseline、deep baseline probe、deep baseline full LOSO。

所以当前阶段的准确说法是：

- `系统已做出来`
- `真实数据链路已打通`
- `自有数据上已有对齐/融合/解释证据`
- `公开数据上已有辅助 benchmark`
- `真正剩下的问题不是“能不能做”，而是“现有证据够不够支撑论文中的两个核心部分”`

## 4. 现有证据是否足够支撑双流对齐

### 4.1 已经足够支撑的部分

当前证据已经足够支撑下面这些结论：

1. 你确实实现了一个 `双流连续对齐` 管线，而不是简单插值拼接。
2. 这个对齐管线在真实 sortie 上可运行，并能稳定产出 `Stage H view` 资产。
3. 物理一致性约束已经真实接入，不是论文中的口头设想。
4. 对齐产物已经能被下游继续消费，不依赖训练期中间对象。

换句话说：

`“双流对齐这部分有没有做出来”这个问题，当前证据已经足够回答“有，而且在自己的数据上跑通了”。`

### 4.2 还不够支撑的部分

当前证据还不足以非常强地支持下面这种更高强度表述：

- “我们的双流对齐在自己的数据集上已经证明是最优的”
- “我们的双流对齐在公开数据上也证明是最优的”

原因有两个：

1. 现在的自有数据验证，重点是 `真实可运行 + 物理一致 + 导出稳定`，不是和大量外部对齐算法做强对抗比较。
2. `UAB / NASA` 并不是“生理流 + 航电流”的完整双流公开数据，它们不能直接拿来证明你的完整双流对齐主管线最优。

### 4.3 对双流对齐最关键的下一步

如果目的是让论文中的“对齐部分”更有说服力，最重要的不是多训练几个 epoch，而是：

1. 在现有 `Stage H` 资产上，再整理一份专门的“对齐支撑报告”。
2. 把 `E baseline`、`E+F(full)`、`Stage H diagnostics` 串起来，形成一条完整证据链：
   - 对齐前后
   - 加不加物理约束
   - 导出后稳定性如何
   - 同 sortie 双 pilot 差异是否合理
3. 如果时间允许，再补一个“弱外部对照”会更强：
   - 例如简单插值/强制对齐/无物理约束路径
   - 目的不是再造一个大系统，而是给论文一个“不是随便对齐一下也能一样好”的对照

所以：

`双流对齐部分，最值得补的是“更直接的对齐对照和汇总”，不是公开数据上继续盲目加 epoch。`

## 5. 现有证据是否足够支撑因果融合

### 5.1 已经足够支撑的部分

当前证据已经足够支撑下面这些结论：

1. 你已经实现了符合选题报告描述的最小因果融合机制：
   - 当前生理状态作为 Query
   - 历史航电事件作为 Key/Value
   - 非对称因果掩码切断逆向信息流
2. 该机制已经在真实 sortie 上运行并导出注意力/事件贡献结果。
3. `Phase 2 case study` 已经证明它至少能做：
   - `same-sortie dual-pilot` 对照
   - `vehicle_delta_suppressed` 等 bundle-only 消融
   - `WARN` view 的主线解释

换句话说：

`“因果融合这部分有没有被实现，并且在自己的数据上给出可解释结果”这个问题，当前证据已经足够回答“有”。`

### 5.2 还不够支撑的部分

当前证据还不足以非常强地支持下面这种更高强度表述：

- “因果融合阶段在自有数据上已经被严格证明优于去掉因果约束的所有替代方案”
- “因果融合阶段在公开数据上也证明是最优的”

主要原因是：

1. `阶段 G(min)` 当前更多是“机制有效 + 可解释输出存在”的证明。
2. `Phase 2` 虽然有真实 sortie 消融，但它主要是 case-study 级证据，还不是大规模标签驱动的任务级 superiority 证明。
3. `UAB / NASA` 公开数据没有真实航电流，不适合直接证明“因果融合主管线”最优。

### 5.3 对因果融合最关键的下一步

如果目的是让论文中的“因果融合部分”更有说服力，最重要的也不是先去加 epoch，而是：

1. 在自有 `Stage H` 资产上，明确形成一组“融合层专属消融”。
2. 最少应强调下面几类对照：
   - `F baseline` vs `F+G(min)`
   - `F+G(min)` vs `F+G(no causal mask)`
   - `F+G(min)` vs `vehicle_delta_suppressed`
   - `F+G(min)` vs `no_event_bias`
3. 若当前没有强标签下游任务，也至少应把以下指标正式化为“因果融合证据”：
   - dual-pilot separability
   - event-mask interference sensitivity
   - attention entropy / top-event concentration
   - `PASS/WARN` view 的一致性解释

所以：

`因果融合部分，最值得补的是“自有数据上的更正式消融”，不是继续在 UAB/NASA 上盲目追分。`

## 6. UAB / NASA 到底能支撑什么，不能支撑什么

### 6.1 能支撑的

它们能支撑：

1. 你在公开 benchmark 上做过认真对照。
2. 你不是只跑了自己方法，还跑了 `MulT / ContiFormer`。
3. 结果说明“更重的深度模型并不会自然全面更优”。
4. `ContiFormer` 在 `UAB subjective regression` 上显示出局部潜力。

### 6.2 不能直接支撑的

它们不能直接支撑：

1. 你的完整 `双流对齐 + 因果融合` 主管线在公开数据上最优。
2. 你的 `因果融合` 阶段在公开数据上被直接验证。
3. 你的 `双流人机异构` 主命题在公开数据上被完整复现。

根本原因是：

- `UAB` 是生理 workload 数据
- `NASA` 是 attention-state / risk proxy 数据
- 它们并不等价于“生理流 + 飞机航电流”的自有任务场景

所以最合理的定位是：

`UAB / NASA 是辅助 benchmark，不是证明主管线最优的主战场。`

## 7. 现在最应该做什么

如果目标是“让论文这两个核心部分更有支撑力”，建议优先级如下：

### 第一优先级：补自有数据的论证闭环

先补这两份面向论文的总结/实证文档：

1. `双流对齐支撑报告`
   - 把 `E`、`F`、`H` 串成一条闭环
   - 重点回答：为什么这不是简单拼接，而是物理一致的连续对齐
2. `因果融合支撑报告`
   - 把 `G(min)`、`H`、`Phase 2 case study` 串成一条闭环
   - 重点回答：为什么这不是普通注意力，而是受物理因果约束的单向融合

### 第二优先级：补最能提升说服力的实验

如果只补一类实验，优先补：

1. `自有数据上的额外消融`
2. `如果能做，再补一个弱外部对齐基线`

而不是优先做：

1. 给所有 deep baseline 统一多训几个 epoch
2. 再去找一堆不完全匹配的新公开数据集

### 第三优先级：只做有希望的调参

如果你还想从公开数据里再挖一点结果，唯一最值得继续的是：

- `ContiFormer + UAB subjective regression`

原因：

1. 这是当前唯一明确优于现有 classical baseline 的辅助结果方向。
2. 它更像是在支撑“连续时间建模有价值”，而不是在浪费算力追一个大概率追不上的 objective 分类。

所以这里可以考虑：

1. `epochs` 稍微增大
2. 只对 `ContiFormer + UAB subjective` 做小范围调参
3. 不建议再优先花时间在 `NASA objective` 或 `UAB objective` 上

## 8. 最终判断

当前最准确的判断是：

1. `双流对齐`：已经有足够证据证明“做出来了，并且在自己的数据上有效”。
2. `因果融合`：已经有足够证据证明“做出来了，并且在自己的数据上能产生可解释结果”。
3. 但如果你想更强地写“这两部分在论文中被充分证明优于替代方案”，当前证据还差一层：
   - `自有数据上的更正式定量消融 / 对照`
4. `UAB / NASA` 已经足够作为辅助 benchmark，但它们不是主管线 superiority 的核心证据来源。

因此最合理的下一步不是“继续广撒网跑实验”，而是：

1. `先补自有数据上的两份支撑性总结/对照文档`
2. `再补最必要的自有数据消融`
3. `最后只对 ContiFormer + UAB subjective 做定向调参（如果还想追一个更漂亮的辅助结果）`
